# 资产架构

  • Web 单个源码指向安全

    单个目录内只存在一个站点

  • Web 多个目录源码安全

    在一个站点目录内存在另一站点

    xxx.com 一个站点

    xxx.com/blog 另一个站点

    一个网站，两个 Web 应用程序，任何一个程序出现漏洞，都可以进入安全测试

  • Web 多个端口源码安全

    一个域名的不同端口对应不同的 Web 应用程序

    xxx.com 一个站点

    xxx.com:8081 另一个站点

    一个网站，两个 Web 应用程序，相当于多个目标

  • 架设第三方插件接口安全

    插件分类：

    • 可以外连的插件，eg：phpmyadmin（方便了管理人员，也方便了攻击者）
    • 仅支持后台使用的插件，eg：编辑器等

    安装的插件越多，面临的安全问题越多

  • 单个服务器架设多个站点安全

    a.xxx.com - 192.168.1.1

    b.xxx.com - 192.168.1.1

    给到目标 a.xxx.com 尝试对 b.xxx.com 进行安全测试

  • 服务器架设多个应用安全

    不只是 Web 应用，还装了服务器应用，eg：ftp 应用（方便了管理人员，也方便了攻击者）

# 番外安全

  • 基于域名解析安全

    对域名管理平台进行社工猜解等攻击，从而劫持域名，修改域名解析，达到攻击的目的

  • 基于服务器本身安全

    服务器连接密码被猜解

  • 基于服务商信息安全

    服务商平台账号密码密码被猜解

  • 基于个人安全

    利用社工等手段，个人电脑被攻击或个人信息被掌握，典型的点对点攻击

# 考虑阻碍

  • 站库分离

    数据库与站点不在一台服务器上

  • CDN 加速服务

    网站在全世界分布节点（站点缓存），就近访问达到加速的目的，同时也掩盖了真实服务器的位置，

  • 负载均衡服务

    用户群体庞大，为防累积，厂商会准备多台服务器，提供服务的可能是备用机，攻陷备用机不一定能达到目的

  • WAF（web 应用防火墙）

    防止一些利用 web 应用漏洞的攻击

    • 普通 waf
    • 云 waf，会自动学习，更智能

  • 主机防护防火墙

    当攻击网站成功，想拿到某些权限，上传后门或木马时，主机防护防火墙可能会报警
